**MonthPickerView.swift:**
- Takes `@Binding var currentMonth: Date` (represents 1st of displayed month)
- HStack: chevron.left Button, Text with "MMMM yyyy" format (e.g. "March 2026"), chevron.right Button
- Chevron buttons: call `Calendar.current.date(byAdding: .month, value: -1/+1, to: currentMonth)` + HapticManager.selection()
- "Today" pill: shown when currentMonth is NOT the current calendar month. Tapping resets to current month. Style: small capsule with "Today" text, accent color background
- The entire view supports .gesture(DragGesture) for swipe: onEnded, if horizontal translation > 50 go next month, < -50 go previous month
- Note: the DragGesture is on MonthPickerView itself (not the full ScrollView — to avoid conflict with scroll). Use .simultaneousGesture if needed on the parent.

**MonthlySummaryCard.swift:**
- Takes: `income: Double`, `totalSpent: Double`
- Computed: `netSaved = income - totalSpent`, `percentUsed = income > 0 ? min(totalSpent / income, 1.0) : 0`
- VStack with .cardStyle():
  - Row: "Income" label + income.formattedEUR (or "Not set" when income == 0)
  - Row: "Total Spent" label + totalSpent.formattedEUR
  - Divider
  - Row: net label + amount. When netSaved >= 0: "Net Saved" green text. When < 0: "Overspent" red text, show absolute value with minus sign
  - Progress bar (only when income > 0): GeometryReader horizontal bar, filled portion = percentUsed. Green when percentUsed < 1.0, red when >= 1.0. Height ~8pt, cornerRadius 4
  - Below progress bar: "\(Int(percentUsed * 100))% of budget used" in .caption .secondary

**DashboardEmptyState.swift:**
- Takes `onAddExpense: () -> Void` closure
- VStack centered: SF Symbol (banknote or tray), "No expenses yet this month" text, "Add your first expense" button that calls onAddExpense
- Button styled with accent color background, white text, capsule shape

**Replace DashboardView.swift stub with full implementation:**

- `@Environment(\.modelContext) private var modelContext`
- `@State private var currentMonth: Date = ...` (initialize to first day of current month using Calendar.current.dateComponents + date(from:))
- `@Query private var allExpenses: [Expense]` — fetch ALL expenses. Filter in-view by monthYear string computed from currentMonth.
  - Compute `currentMonthYear: String` from currentMonth using DateFormatter "yyyy-MM"
  - `let monthExpenses = allExpenses.filter { $0.monthYear == currentMonthYear }`
  - This avoids dynamic @Query predicate issues — all filtering is in-memory (fine for personal budget app volume)
- Fetch MonthlyBudget: use `MonthlyBudget.fetchOrCreate(monthYear: currentMonthYear, in: modelContext)` to get income. Store result in computed or use a helper.
